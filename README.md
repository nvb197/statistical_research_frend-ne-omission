# The Omission of *ne* in French 

Code and derived data for the statistical report *The Omission of ne in French:
Register, Sub-register, and Internal Conditioning in the FReND Corpus* (Nguyen Van Bach,
Université Paris-Dauphine–PSL / Lattice, 2026).

The report measures how often French drops the preverbal clitic *ne* from its bipartite
negation (*ne… pas* → *pas*) across three corpus genres, and asks what conditions that
omission. Every number, table and figure in the report is regenerated by the scripts here;
none is entered by hand.

## Quick start

```bash
git clone https://github.com/nvb197/statistical_research_frend-ne-omission.git
cd statistical_research_frend-ne-omission
pip install -r requirements.txt

python phase1_stats.py             # descriptive counts, densities, typology
python phase2_ne.py                # ne-omission and ne littéraire
python phase3_bivarie.py           # bias-corrected Cramér's V, corpus vs unit
python phase4_inferentiel.py       # Wilson CIs, odds ratios, Kruskal–Wallis, residuals
python phase5_mixtes.py            # omission GLMM, crossed GLMM, log-length LMM, density GEE
python modeles_complementaires.py  # tense-adjusted GLMM, prior sensitivity, regrouped typology + LMM
python figures_rapport.py          # the six figures of the report
```

The scripts read the Parquet files sitting beside them and write CSVs to
`resultats_phase1/` … `resultats_phase5/`, and PDFs to `figures/`. No configuration, no
paths to set. Run them in the order above: `figures_rapport.py` needs
`resultats_phase5/modele_A_chute.csv`, and `modeles_complementaires.py` is easiest to read
after `phase5_mixtes.py`.

Most phases take seconds. The mixed models (`phase5_mixtes.py`,
`modeles_complementaires.py`) take roughly 10–25 minutes: the variational-Bayes fits are the
slow part.

Rebuilding the Parquet files **from the corpus XML** additionally requires the FReND release
and spaCy; see [Rebuilding from the corpus](#rebuilding-from-the-corpus).

## Layout

The scripts resolve data by bare filename, so everything lives in one directory. Outputs are
regenerable and are not versioned.

```
negations.parquet              one row per negation (8,957)
denominateurs.parquet          one row per document part (6,666, incl. 4,020 with no negation)
phrases_par_document.parquet   one row per document (6,354)

expletifs.py                   the three confirmed expletive ne — single source of truth
clean_xml_corpus.py            curation of the raw XML export (5 ordered operations)
corriger_xml.py                strips the expletive tags from the XML (text untouched)
build_negations_master.py      corrected XML  → negations.parquet
denominateurs.py               corrected XML  → denominateurs.parquet, phrases_par_document.parquet

phase1_stats.py                → resultats_phase1/
phase2_ne.py                   → resultats_phase2/
phase3_bivarie.py              → resultats_phase3/
phase4_inferentiel.py          → resultats_phase4/
phase5_mixtes.py               → resultats_phase5/
modeles_complementaires.py     → resultats_phase5/  (4 further CSVs)
figures_rapport.py             → figures/
```

## Pipeline

```
  FReND XML   (not redistributed — obtain from the corpus authors)
      │
      │  clean_xml_corpus.py     5 ordered curation operations,
      ▼                          one CSV audit row per applied change
  my_xml_corpus_clean.xml  +  cleaning_log.csv
      │
      │  corriger_xml.py         removes the 3 confirmed expletive ne
      ▼                          (tags only — the text is left intact)
  my_xml_corpus_clean_corrige.xml
      │
      ├─ build_negations_master.py ─→ negations.parquet
      └─ denominateurs.py ──────────→ denominateurs.parquet, phrases_par_document.parquet
      │
      ├─ phase1_stats.py        descriptives, densities, typology
      ├─ phase2_ne.py           omission, ne littéraire
      ├─ phase3_bivarie.py      Cramér's V, corpus level vs unit level
      ├─ phase4_inferentiel.py  Wilson, odds ratios, Kruskal–Wallis, residuals
      └─ phase5_mixtes.py       the mixed models
             └─ modeles_complementaires.py
                    └─ figures_rapport.py
```

`expletifs.py` is a data module, not a step. It holds the single authoritative list of the
three confirmed expletive *ne* (*je crains qu'il **ne** vienne* — negative in form, not in
meaning). `corriger_xml.py` uses it to strip their tags from the XML; phases 1–5 filter on
the same list in memory, as a safety net for the case where the Parquet was not rebuilt from
the corrected XML. One list, so the two routes cannot diverge, and the count is 8,957 either
way.

## The models of the report

Named as in the report. `phase5_mixtes.py` fits the first four,
`modeles_complementaires.py` the rest.

| Report name | Specification | Output CSV |
|---|---|---|
| omission GLMM | Bayesian logistic; fixed: corpus + negator; random: work | `modele_A_chute.csv` |
| crossed extension | adds a crossed random intercept per verb lemma | `modele_A_chute_verbe.csv` |
| crossed + tense | adds tense as a fixed effect (undefined tense kept as its own level) | `modele_croise_temps.csv` |
| log-length LMM | Gaussian, REML; fixed: corpus + surface configuration; random: work | `modele_B_regroupe.csv` |
| density GEE | Poisson, marginal; offset log(tokens); clusters: work (sandwich SE) | `modele_C_densite.csv` |
| — (sensitivity) | omission GLMM refit at prior SD ∈ {1, 2, 3, 5} | `sensibilite_prior.csv` |

Two things worth knowing before reading the code.

**Why the omission model is Bayesian.** `sequoia` omits at ≈0%, a quasi-complete separation
under which maximum likelihood sends the coefficient to −∞. The weakly informative Gaussian
prior (SD 2) supplies the regularisation that the likelihood alone cannot. The prior is not
innocuous, and `sensibilite_prior.csv` says by how much: the register odds ratio moves from
12.1 to 18.5 across prior SD 1→5, against 16.5 at the reported SD of 2.

**`modele_B_regroupe.csv` supersedes `modele_B_portee.csv`.** The latter used the
annotation's native class scheme, which pools bare *ne* with bare second elements — two
configurations the report keeps apart, and which turn out to pull scope length in opposite
directions (0.76× vs 1.16×). Pooling them controls for neither. The report uses the regrouped
refit; `classe_surface()` in `modeles_complementaires.py` is the single definition of that
classification, shared by the typology table and the model.

## Data

**The FReND corpus is not redistributed here.** It is a separate resource
(Le Cloirec-Ait Yahya, Seminck & Amsili 2024), released under CC BY-NC-SA 4.0; obtain it from
its authors.

What is here is the **derived** annotation: one row per negation, with genre, unit, negator,
surface configuration, verb lemma, tense, scope length in tokens, and so on. The column
holding the scope **text** is deliberately excluded, for two reasons:

1. the blog subcorpus consists of first-person testimonies of workplace sexism, and this study
   uses it only for aggregate analysis of grammatical variation — republishing text fragments
   would go beyond that purpose;
2. nothing needs it. Every statistic in the report is computed from the structural columns,
   and the pipeline runs unchanged without the text.

Anyone holding the FReND release can rebuild the full table, text column included, with
`build_negations_master.py`.

Two counting grains are kept in separate tables on purpose. `negations.parquet` has one row
per negation; `denominateurs.parquet` has one row per document part, **including the 4,020
parts that contain no negation at all**. Density joins the two. Using the negation table as
its own denominator would silently drop those parts and inflate every rate.

### Rebuilding from the corpus

```bash
# with the FReND XML in place, named my_xml_corpus.xml
python clean_xml_corpus.py       # → my_xml_corpus_clean.xml + cleaning_log.csv
python corriger_xml.py           # → my_xml_corpus_clean_corrige.xml
# set XML_PATH in build_negations_master.py to the *corrected* file, then:
python build_negations_master.py # → negations.parquet
python denominateurs.py          # → denominateurs.parquet, phrases_par_document.parquet
```

The curation touches annotation tags only; the underlying text is byte-identical, so token
counts and density denominators are unaffected. `cleaning_log.csv` records one row per applied
change. The arithmetic closes: 8,958 cue-counted negations in the resource paper, +2
mislabelled *non* retyped as cues, −3 expletives, = 8,957 analysed instances.

## Requirements

```
pandas>=2.0  numpy>=1.24  scipy>=1.10  statsmodels>=0.14  pyarrow>=12.0  matplotlib>=3.7
lxml>=4.9        # corriger_xml.py only
spacy==3.8.14    # rebuilding from XML only, with the fr_core_news_md 3.8.0 model
```

```bash
pip install -r requirements.txt
python -m spacy download fr_core_news_md   # only if rebuilding from XML
```

Token counts are rule-based and stable across spaCy versions; sentence boundaries and verbal
morphology come from the parse and are pinned to the version above. Permutation tests use a
fixed seed and the variational-Bayes fits are deterministic given the data, so repeated runs
produce identical output.

## Notes for readers of the code

Comments and identifiers are in French, matching the corpus and the host laboratory; the
report is in English.

`phase4_inferentiel.py` also computes analyses that were dropped from the final report (verb
mood, negation type by genre). They are left in place: they run, they are correct, and they
record what was explored. The report's claims come from the scripts and outputs listed above.
